# ZIP Import Feature Guide

This document explains the new ZIP import feature that allows users to import doujin/manga ZIP files directly into the app's local folder.

## ЁЯОп Feature Overview

The ZIP import feature enables users to:
- Select ZIP files using the native Android file picker
- Automatically extract images to a dedicated 'local' folder
- Auto-generate metadata for immediate viewing
- Have imported content registered in the database
- View imported content immediately in the app

## ЁЯУБ Folder Structure

Imported ZIP files are extracted to the 'local' source folder, separate from other sources:

```
[Storage Root]/nhasix/local/[content-id]/
тФЬтФАтФА images/
тФВ   тФЬтФАтФА 001.jpg
тФВ   тФЬтФАтФА 002.webp
тФВ   тФФтФАтФА ...
тФФтФАтФА metadata.json (auto-generated)
```

**Key Points:**
- **Separate from nhentai:** Local imports go to `/nhasix/local/`, not `/nhasix/nhentai/`
- **Content ID:** Sanitized from ZIP filename (removes .zip, special chars)
- **Auto-metadata:** metadata.json is automatically created

## ЁЯЪА User Guide

### How to Import a ZIP File

1. **Open Offline Content Screen**
   - Tap hamburger menu (тШ░)
   - Select "Offline Content" or "Downloaded"

2. **Tap Import ZIP Button**
   - Look for the folder_zip icon (ЁЯУБ) in the app bar
   - This is next to the Import and Export buttons

3. **Select ZIP File**
   - Native Android file picker opens
   - Browse and select your ZIP file
   - Tap "Open" or "Select"

4. **Wait for Extraction**
   - App shows loading state
   - ZIP is extracted to local folder
   - Metadata is auto-generated
   - Content is registered in database

5. **View Your Content**
   - Content appears in Offline Content list immediately
   - Source shows as "local"
   - Title derived from ZIP filename

### Supported ZIP Structures

The app supports flexible ZIP structures:

**Option 1: Images in root**
```
my-doujin.zip
тФЬтФАтФА 001.jpg
тФЬтФАтФА 002.jpg
тФФтФАтФА 003.jpg
```

**Option 2: Images in subfolder** (recommended)
```
my-doujin.zip
тФФтФАтФА images/
    тФЬтФАтФА 001.webp
    тФЬтФАтФА 002.webp
    тФФтФАтФА 003.webp
```

**Option 3: Mixed structure**
```
my-doujin.zip
тФЬтФАтФА cover.jpg
тФЬтФАтФА 001.jpg
тФЬтФАтФА 002.jpg
тФФтФАтФА some-folder/
    тФФтФАтФА ignored-file.txt
```

**Supported Image Formats:**
- `.jpg` / `.jpeg` тЬЕ
- `.png` тЬЕ
- `.webp` тЬЕ (including the user's case!)
- `.avif` тЬЕ
- `.gif` тЬЕ
- `.bmp` тЬЕ

Non-image files are ignored during extraction.

## ЁЯУЭ Auto-Generated Metadata

When you import a ZIP without metadata, the app creates:

```json
{
  "id": "sanitized-zip-name",
  "sourceId": "local",
  "source": "local",
  "title": "Sanitized Zip Name",
  "coverUrl": "file:///path/to/first/image.jpg",
  "totalImages": 42,
  "downloadedAt": "2026-03-31T10:00:00.000Z",
  "schemaVersion": 2,
  "autoGenerated": true,
  "importedAt": "2026-03-31T10:00:00.000Z",
  "importedFrom": "zip",
  "originalFileName": "my-doujin.zip"
}
```

**Fields Explained:**
- `id`: Sanitized ZIP filename (lowercase, hyphens)
- `sourceId`: Always "local" for ZIP imports
- `title`: Formatted title from ZIP name
- `coverUrl`: First image found in ZIP
- `totalImages`: Count of extracted images
- `autoGenerated`: Flag indicating auto-created metadata
- `importedFrom`: Always "zip" for this feature
- `originalFileName`: Original ZIP filename

## ЁЯРЫ Troubleshooting

### "Import failed: No images found in ZIP file"
- **Cause:** ZIP contains no supported image files
- **Solution:** Ensure ZIP has .jpg, .png, .webp, etc. files

### "Import failed: Failed to read ZIP file"
- **Cause:** Permission denied or invalid file
- **Solution:** Grant storage permission, ensure valid ZIP

### "Imported content not showing"
- **Cause:** UI not refreshed
- **Solution:** Pull to refresh or restart app

### Content shows but can't open
- **Cause:** Corrupted extraction
- **Solution:** Delete and re-import

### ZIP picker doesn't open
- **Cause:** Native handler not registered
- **Solution:** Rebuild app, check logs

## ЁЯТб Best Practices

1. **ZIP Naming:**
   - Use descriptive filenames
   - Avoid special characters
   - Example: `my-manga-vol-01.zip` тЖТ Title: "My Manga Vol 01"

2. **ZIP Structure:**
   - Keep images in `images/` subfolder
   - Use sequential numbering (001.jpg, 002.jpg, ...)
   - Avoid nested folders

3. **Image Quality:**
   - Use compressed formats (webp recommended)
   - Keep file sizes reasonable
   - Maintain consistent dimensions

4. **Organization:**
   - One doujin per ZIP
   - Clear, unique filenames
   - No duplicate imports

## ЁЯЖЪ Comparison: Import ZIP vs Import from Backup

| Feature | Import ZIP | Import from Backup |
|---------|-----------|-------------------|

- Content URI permissions granted temporarily
- Persistent permission attempted but not required
