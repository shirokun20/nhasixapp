Base URL: https://hentai-cosplay-xxx.com

Technology: AMP (Accelerated Mobile Pages)

1. HOME PAGE Structure

URL: / or /page/{number}/

❌ Current Config (WRONG)

"container": ".item" // ❌ DOES NOT EXIST "title": { "selector": ".title" } // ❌ DOES NOT EXIST

📄 Actual HTML Structure

<!-- Item Container: Direct <a> tags --> <a href="https://hentai-cosplay-xxx.com/image/meenfox-elegg-1/"> Meenfox – Elegg 1 </a> <a href="https://hentai-cosplay-xxx.com/image/meenfox-elegg-1/"> Meenfox – Elegg 1 </a> 2026/05/16 <!-- Pagination --> <a href="/page/2/">Next ></a> <a href="/page/18026/">Last >></a>

✅ Recommended Fix

"container": "a[href*='/image/']", // ✅ Select all links to /image/ "fields": { "id": { "selector": "", // Empty = use container itself "attribute": "href", "transform": "slug" }, "title": { "selector": "", // Empty = use container itself "attribute": "textContent" // Get link text }, "coverUrl": { "selector": "img, amp-img", "attribute": "data-src,src" // Check data-src first (lazy load) } }
⚠️ Note: Images use AMP lazy loading. They may have data-src instead of src initially.

4. READER PAGE Structure

URL: Same as detail page (/image/{id})

❌ Current Config (FRAGILE)

"selector": "amp-img[src*=upload]:not(.related-thumbnail)" // ❌ Too specific, filters by URL pattern

📄 Actual HTML Structure

<!-- Images are in <a> tags, NOT <img> tags! --> <a href="https://static17.hentai-cosplay-xxx.com/upload/.../1.jpg"> Meenfox – Elegg 1 1th image </a> <a href="https://static17.hentai-cosplay-xxx.com/upload/.../2.jpg"> Meenfox – Elegg 1 2th image </a> <a href="https://static17.hentai-cosplay-xxx.com/upload/.../3.jpg"> Meenfox – Elegg 1 3th image </a>

✅ Recommended Fix (Extract from links)

"reader": { "container": "body", "images": { "selector": "a[href*='/upload/'][href$='.jpg'], a[href*='/upload/'][href$='.png']", "attribute": "href" // ✅ Get image URL from link href } }
💡 Key Insight: Images are stored in <a> tag href attributes, not in <img> tags!
