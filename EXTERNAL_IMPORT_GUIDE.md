# Guide: Importing External Doujins/Manga

This guide explains how to import doujins or manga downloaded from external sources (e.g., imhentai, nhentai web, etc.) into Kuron App.

## Problem

If you manually place a downloaded doujin folder in the app's storage directory (e.g., `/nhasix/nhentai/[folder-name]/`), you might encounter a **"Rendering Error please restart app"** message when trying to view it.

**Root Cause:** The app doesn't know about manually placed content because it's not registered in the internal database.

## Solution

The app provides a built-in **Import** feature that automatically detects and registers manually placed content.

### Step-by-Step Instructions

1. **Place Your Downloaded Content**
   - Put your downloaded doujin folder in the correct location:
     ```
     [Storage Root]/nhasix/[source-name]/[folder-name]/
     ```
   - Example for nhentai:
     ```
     /storage/emulated/0/[Your Custom Storage]/nhasix/nhentai/my-doujin/
     ```
   - The folder should contain:
     - `images/` subdirectory with your image files (`.jpg`, `.png`, `.webp`, `.avif`, etc.)
     - OR images directly in the folder (legacy structure)

2. **Open Offline Content Screen**
   - Tap the **hamburger menu** (☰) in the app
   - Select **"Offline Content"** or **"Downloaded"**

3. **Import the Content**
   - In the Offline Content screen, tap the **Import** button in the app bar
   - The app will automatically:
     - Scan all folders in `/nhasix/`
     - Detect manually placed content
     - **Auto-generate `metadata.json`** for folders that don't have one
     - Register the content in the database
     - Show a progress notification

4. **View Your Content**
   - After import completes, your manually placed doujin will appear in the offline content list
   - You can now open and read it normally

## Auto-Generated Metadata

**NEW in this update:** The app now automatically creates a `metadata.json` file for manually imported content. This file includes:

```json
{
  "id": "folder-name",
  "sourceId": "nhentai",
  "source": "nhentai",
  "title": "Folder Name",
  "coverUrl": "/path/to/first/image.jpg",
  "totalImages": 42,
  "downloadedAt": "2026-03-31T08:00:00.000Z",
  "schemaVersion": 2,
  "autoGenerated": true,
  "importedAt": "2026-03-31T08:00:00.000Z"
}
```

This ensures your content works properly even without original metadata from the source.

## Supported Image Formats

The app supports the following image formats:
- `.jpg` / `.jpeg`
- `.png`
- `.gif`
- `.webp` ✅ (Issue mentioned webp format - fully supported!)
- `.avif`
- `.bmp`

        ├── images/

## Troubleshooting

### "Still getting rendering error after import"
- Make sure you tapped the **Import** button in the Offline Content screen
- Check that your images are in a supported format
- Verify the folder structure matches one of the examples above

### "Import button doesn't find my content"
- Ensure the folder is in `/nhasix/[source-name]/` not just `/nhasix/`
- Supported source names: `nhentai`, `crotpedia`, `komiktap`, `hitomi`, `ehentai`, `hentainexus`
- For unknown sources, use `nhentai` as the source folder

### "Content shows but has wrong title"
- The app uses the folder name as the title
- To set a custom title, create a `metadata.json` file manually before importing
- Or rename the folder to your desired title

## Technical Details

When you import external content:
1. App scans `/nhasix/` recursively
2. Detects folders with images
3. Creates `metadata.json` with folder name as title
4. Registers content in SQLite database with `DownloadState.completed`
5. Refreshes UI to show imported content

The `metadata.json` file acts as a "decoder key" for the app to understand the content, especially when using hashed folder names (Elegant ID system).

## For Advanced Users

If you want to create metadata manually before importing, use this template:

```json
{
  "schemaVersion": "2.0",
  "source": "nhentai",
  "id": "your-content-id",
  "sourceId": "nhentai",
  "title": "Your Custom Title",
  "language": "english",
  "totalImages": 42,
  "downloadedAt": "2026-03-31T08:00:00.000Z"
}
```

Save as `metadata.json` in your content folder before importing.
